# User Story 1.1: Corrigir Fluxo de Confirmação de E-mail

## Status: Draft
**Prioridade:** Crítica 💎
**Responsável:** @dev

## Contexto
O e-mail de confirmação do Supabase não está chegando para as novas empresárias. Isso impede o acesso ao Catálogo Lapidado. A suspeita é de bloqueio por SPAM ou limites do SMTP padrão do Supabase.

## Objetivos
1. Configurar um provedor de e-mail confiável (Resend recomendado).
2. Ajustar a UI da página de registro (`src/app/register/page.tsx`) para mostrar claramente o sucesso do envio.
3. Garantir que o link de ativação funcione corretamente.

## Critérios de Aceitação (AC)
- [ ] O e-mail de confirmação deve chegar na caixa de entrada (ou promoções) do usuário em menos de 1 minuto.
- [ ] A página de registro deve exibir uma mensagem de "E-mail Enviado" estilizada (não em vermelho de erro) após o `signUp`.
- [ ] Adicionar um botão de "Reenviar e-mail de confirmação" na tela de sucesso.
- [ ] O link no e-mail deve redirecionar para `/auth/callback` e validar o usuário.

## Detalhes Técnicos (River's Dev Notes)
- **Supabase SMTP:** O `@dev` deve orientar a Angela a configurar o SMTP no Dashboard do Supabase (Settings > Auth > SMTP) usando as credenciais do **Resend**.
- **UI:** Mudar a lógica em `register/page.tsx` para usar um estado `waitingConfirmation` em vez de setar o erro com a mensagem de "QUASE LÁ".
- **Dependências:** Instalar `resend` se for necessário um envio via API, mas o SMTP do Supabase é o caminho mais rápido agora.
